# カフェECカリキュラム 内容作成 Q&A

スコープ確定のための質問。`A:` の行に回答を書いてください。選択肢がある場合は採用する選択肢のアルファベットだけ書くか、自由記述でかまいません。

---

## 前提の現状認識

- 既存のシードで、`cafe-chapter-0`（15ステージ）と `cafe-chapter-1`（14ステージ）が DB に入った
- 各ステージの description には、仕様書（curriculum-design.md）の「学ぶこと／実習／感動ポイント／Phase」を Markdown で転記してある
- ステージをクリックすると StageContentDialog が開き、description が Markdown で表示される
- 「完了する」ボタンで進捗記録される
- 実習コードエディタを伴う学習ワークスペースは `/learn/hero-section` だけが存在（サンプル）

## Q1. 「内容を作る」のレベル感

どこまでやれば「内容作成完了」とみなしますか。

- (A) ステージのダイアログに表示される Markdown を肉付けするだけ（コードサンプル、詳しい手順、躓きポイント、スクショ置き場のプレースホルダーまで書き込む）
- (B) (A) に加えて、HTML/CSS を実際に書くレッスンについては `/learn/xxx` のような学習ワークスペース（エディタ + プレビュー + ステップガイド）を用意する
- (C) (B) に加えて、座学レッスン（VS Code のインストール、GitHub アカウント作成など実習コードなし）も専用の読み物ページにする
- (D) その他（自由記述）

A:C

## Q2. スコープ分割

29 レッスン全部を一度に作ると巨大になります。どの粒度で PR を切りますか。

- (a) 全部を1 PR で一気に出す
- (b) 「第0章全部」→「第1章全部」の2 PR に分ける
- (c) 「全レッスンの description 肉付け (A 相当)」→「実習ワークスペース (B 相当) を Phase 1 のレッスンから順番に」という段階PR
- (d) まず1レッスンだけプロトタイプして、設計が固まってから量産
- (e) その他（自由記述）

A:d

## Q3. 学習ワークスペースが必要な場合の仕様（Q1 が B または C の場合のみ）

### Q3-1. ルーティング

- (a) 既存の `/learn/[chapterId]` を流用。chapterId に `cafe-ch0-05-first-html` 形式の slug を使う
- (b) `/learn/[curriculumSlug]/[stageNumber]` のような新ルートを追加
- (c) `/curriculum/[slug]/[stageNumber]` のようにカリキュラムページ配下に置く
- (d) その他

A:a

### Q3-2. 実習のない座学系レッスン（VS Code インストール、GitHub アカウント作成など）の扱い

- (a) ワークスペースは作らず、ステージダイアログの Markdown で完結させる
- (b) 読み物ページ（エディタなし、Markdown 本文 + 「完了する」ボタンだけ）を用意する
- (c) その他

A:b

### Q3-3. レッスンクリア判定

- (a) 既存 `/learn/hero-section` と同じく、ユーザーが書いた HTML/CSS を正規表現や DOM 検査で判定
- (b) 「完了する」ボタンを押したら完了扱い（判定しない）
- (c) レッスンごとに使い分ける（例: 実習系は判定あり、読み物は完了ボタンのみ）

A:a
