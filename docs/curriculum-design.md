# コーヒーショップECサイト学習カリキュラム 設計ドキュメント

最終更新: 2026年5月6日

---

## 🎯 カリキュラムの全体像

### 目的

未経験者が **コーヒーショップのECサイト** を1つ作り切ることで、HTML/CSS/JavaScript の基礎を実戦的に身につける。

### 対象者

- プログラミング完全未経験者
- デザイン（Figma）は読める前提ではなく、読み方から教える

### 最終成果物

コーヒーショップのECサイト（複数画面・PC/SP対応）

### 進め方の基本ルール

- **Phase 1（スキル学習）**: 仮のデザインで技術を練習
- **Phase 2（実装）**: Figma のデザインを元に実装
- レッスンごとに明確なゴールと実習を設ける
- 要所で **Pull Request** を作成してレビューを受ける

### 全体の章構成

- **第0章**: 導入フェーズ（環境構築 + HTML/CSS超入門）
- **第1章**: ホーム画面を作ろう
- **第2章**: 詳細画面を作ろう（予定）
- **第3章**: カート画面を作ろう（予定）
- **第4章**: 会員登録画面を作ろう（予定）
- **第5章**: お問い合わせ画面を作ろう（予定）
- **第6章**: ヘッダー/フッターの実装・全画面統合（予定）

---

## 共通ルール

### コーディング規約

- **クラス命名**: kebab-case（例: `.product-card`, `.button-bar`）
- **ファイル構成**: `index.html` + `style.css` のシンプル構成
- **アイコン**: Figma から SVG でエクスポート（アイコンフォント不使用）
- **ハンバーガーメニュー**: 画像ではなく CSS で実装

### 画像アセット

- 商品写真・ヒーロー画像・ロゴ等は Figma からエクスポート
- エクスポートは **各章で都度** 行う（1章最初に全章分をまとめない）
- 練習用の画像には `https://picsum.photos/` を利用

### ブランチ / PR 運用

- 各章で複数回の Pull Request を作成
- PR は学習の節目で、講師がレビュー

# 🌱 第0章: 導入フェーズ（確定版）

## 章のゴール

- 開発環境を整える
- HTML/CSS の「書いたら動く」体験を得る
- GitHub で最初の Pull Request を出せるようになる

## レッスン構成（全15レッスン）

### レッスン 0-1: VS Code のセットアップ

- **学ぶこと:** インストール（Win/Mac両対応）/ 日本語化
- **実習:** インストール + 起動確認

### レッスン 0-2: VS Code の使い方

- **学ぶこと:** フォルダを開く / 新規ファイル / 保存 / 拡張機能のインストール方法
- **実習:** フォルダを開いてファイルを作ってみる

### レッスン 0-3: Prettier を設定しよう

- **学ぶこと:** フォーマッターとは / Prettier拡張機能 / 保存時自動整形
- **実習:** `.prettierrc` をコピペで配置（コメント付きで設定内容を細かく説明）/ 実際に整形される体験
- **感動ポイント:** 「勝手にコードが綺麗になった！」
- **備考:** HTML/CSS/JS すべてに Prettier を適用する想定

### レッスン 0-4: ターミナルに慣れよう

- **学ぶこと:** ターミナル（Mac）/ コマンドプロンプト（Win）とは / 基本コマンド（cd / ls / pwd / mkdir）
- **実習:** フォルダ移動、作成などの基本操作

### レッスン 0-5: はじめての HTML

- **学ぶこと:** HTMLとは / `.html` ファイルはブラウザで開ける
- **実習:** `hello.html` を作って `<h1>こんにちは</h1>` を表示
- **感動ポイント:** 「自分の書いた文字がブラウザに出た！」

### レッスン 0-6: 色んなタグを試そう

- **学ぶこと:** h1〜h6 / p / br / strong / em / ul / ol / li / a / img / div / span
- **実習:** 自己紹介ページを作ってみる

### レッスン 0-7: HTMLの基本ルール

- **学ぶこと:**
  - `<!DOCTYPE html>`
  - html / head / body の役割
  - `<meta charset="UTF-8">` / `lang="ja"`
  - `<title>`
  - `<meta name="viewport">`
  - タグの入れ子ルール
  - HTMLコメント `<!-- -->`
- **実習:** 0-6で作ったファイルを「ちゃんとした形」に整える

### レッスン 0-8: Chrome の開発者ツールを使ってみよう

- **学ぶこと:** F12で開く / Elements パネル / Console パネル / デバイスモード
- **実習:** 開発者ツールで自分のHTMLを調査

### レッスン 0-9: CSS で装飾してみよう

- **学ぶこと:**
  - CSSとは / style属性 / `<style>`タグ / 外部CSSファイル
  - 要素セレクタ
  - `color` / `background-color` / `font-size` / `font-weight` / `text-align` / `line-height` / `width` / `height` などの基本プロパティ
- **実習:** 自己紹介ページに色とフォントサイズをつける
- **感動ポイント:** 「文字が青くなった！」

### レッスン 0-10: クラスセレクタを使おう

- **学ぶこと:** class属性 / クラスセレクタ / クラス名のつけ方（kebab-case）
- **実習:** 複数の要素に別々のスタイルをクラスで当て分け

### レッスン 0-11: リセットCSSを入れよう

- **学ぶこと:** ブラウザのデフォルトスタイル / リセットCSSとは / 教育用の簡易リセットCSSを自分で書く
- **実習:** リセットCSSを書いて適用、素の状態を確認

### レッスン 0-12: メディアクエリって何？

- **学ぶこと:** PC表示とスマホ表示の違い / `@media` の基本 / DevToolsのデバイスモード再訪
- **実習:** 「画面幅が狭いと文字色が変わる」ミニ実験

### レッスン 0-13: GitHub と Git を準備しよう

- **学ぶこと:**
  - GitHubとは / Gitとは
  - GitHubアカウント作成
  - Gitインストール（Win/Mac両対応）
  - Git Credential Manager
  - 初期設定（`git config --global user.name / user.email`）
- **実習:** アカウント作成 + Git インストール確認

### レッスン 0-14: 学習用リポジトリを準備しよう

- **学ぶこと:** テンプレートリポジトリ / clone
- **実習:** テンプレートから自分のリポジトリ作成 → clone
- **備考:** リポジトリ構成の詳細は `README.md` を参照と案内

### レッスン 0-15: 最初の Pull Request

- **学ぶこと:** branch / add / commit / push / PR の超基本
- **実習:** 自己紹介ページを branch切って commit → push → PR作成
- **📮 卒業PR:** 第0章修了PR

# 🏠 第1章: ホーム画面を作ろう（確定版）

## 章のゴール

- Webページ制作の基本スキルを一通り使えるようになる
- ホーム画面 PC版・SP版を完成させる

## デザインの把握

### 画面構成（上から順に）

1. **ヘッダー**（本章ではプレースホルダー、第6章で実装）
   - PC: ロゴ左 / ナビ右（カート一覧・お問い合せ）/ 右端に検索・カート一覧ボタン
   - SP: ロゴ左 / ハンバーガーアイコン右
2. **MainView**: カフェの背景写真 + 中央に白い見出しボックス「コーヒーショップを作ってみよう！」
3. **ButtonBar**: 黒い帯に「商品一覧」見出し + 並び替えドロップダウン
4. **ProductLists**: 商品カードグリッド（PC: 4列×2行 / SP: 1列×8行）
5. **CTAブロック**: ダークグレー背景に案内文 + 「カート一覧を見る」ボタン（白pill形）
6. **フッター**（本章ではプレースホルダー、第6章で実装）: オレンジ背景 + テキスト

### カラーパレット（目安）

- メインオレンジ: ヘッダー/フッター
- ベージュ: ページ背景
- 黒: ButtonBar
- ダークグレー: CTAブロック
- 白: カード背景

### 特徴

- 角丸多め
- 影は控えめ
- 商品写真・ヒーロー画像が主役

## 📚 Phase 1: スキル学習

### レッスン 1-1: デザインを読み解こう

- **学ぶこと:**
  - Figmaの基本操作
  - 画像のエクスポート方法（SVG / PNG）
  - 色・フォント・サイズの読み取り方
- **実習:** ホーム画面のデザインを観察 / 必要な画像を `assets/images/` にエクスポート

### レッスン 1-2: フォントを読み込んで使おう

- **学ぶこと:**
  - `@import` でフォント読み込み
  - `font-family` / `font-weight` / `font-size`
  - フォント系プロパティの基本
- **実習:** 指定フォントを読み込んでテキストに適用

### レッスン 1-3: 画像を表示しよう

- **学ぶこと:**
  - `<img>` タグ
  - `object-fit` / `object-position`
  - `background-image` / `background-size` / `background-position`
  - `<img>` と `background-image` の使い分け
- **実習:** `picsum.photos` の画像で色々試す

### レッスン 1-4: ボックスモデルを理解しよう

- **学ぶこと:**
  - `margin` / `padding` / `border`
  - `box-sizing: border-box`
  - `display: block / inline / inline-block`
- **実習:** 同じ見た目を違うボックス設定で作り比べる

### レッスン 1-5: コンテナで中央寄せしよう

- **学ぶこと:**
  - `max-width` + `margin: 0 auto`
  - `width` と `max-width` の使い分け
- **実習:** コンテナを作って中に要素を配置

### レッスン 1-6: Flexboxで横並びを作ろう

- **学ぶこと:**
  - `display: flex`
  - `justify-content` / `align-items` / `gap`
  - `flex-direction`
  - Flexの使いどころ（1次元レイアウト）
- **実習:** 「左に見出し・右に要素」のバー状レイアウトを作る

### レッスン 1-7: Gridで格子レイアウトを作ろう

- **学ぶこと:**
  - `display: grid`
  - `grid-template-columns` / `gap`
  - `repeat()` / `1fr`
  - Gridの使いどころ（2次元レイアウト）
  - **FlexとGridの使い分け指針**
- **実習:** 4列のカードグリッドを作る

### レッスン 1-8: ホバーで動きをつけよう ⭐

- **学ぶこと:** `:hover` / 色の変化
- **実習:** ボタンにホバーで色変化をつける
- **感動ポイント:** 「動いた！」
- **備考:** トランジション等の高度な話には触れない

### レッスン 1-9: フォーム要素をカスタマイズしよう

- **学ぶこと:**
  - `<input type="text">` / `<button>` / `<select>` + `<option>`
  - ブラウザデフォルトスタイルの問題点
  - `appearance: none` でリセット
  - padding / border / border-radius でのカスタマイズ
  - `<button>` へのホバー適用
  - `<select>` の矢印をSVGで差し替え（`background-image`）
- **実習:** 仮のフォーム（名前入力・カテゴリ選択・送信ボタン）を一からデザイン
- **備考:** `:focus` は扱わない

# 📝 設計上の重要な決定事項メモ

## 教育方針

- **「書いたら動く」体験を早めに作る**
- 各レッスンに **感動ポイント** を設計
- **Phase 1 = 仮の題材で技術練習**、**Phase 2 = 本番デザインで実装** の二段構え

## 技術スコープの線引き

- **第0章ではCSS変数を扱わない**（必要性が出る場面まで遅延）
- **アイコンフォント不使用**（SVGで統一）
- **ハンバーガーメニューはCSS実装**
- **BEMは採用せず、kebab-case のクラス命名のみ**
- **CSS分割はせず `style.css` 1ファイル運用**

## 今後の緻密化で確認するべき論点

- 第2章での JavaScript 導入タイミング
- 各章の PR 回数の適正化
- 全章を通した「感動ポイント」の配置バランス
