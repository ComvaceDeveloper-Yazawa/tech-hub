# カリキュラム学習フロー 画面仕様書

## 画面遷移図

```
/curriculum (カリキュラム選択)
    │
    │ 本棚の背表紙をクリック → 本が開くアニメーション → 「冒険を始める」ボタン
    │
    ▼
/curriculum/[slug] (ステージ選択)
    │
    │ ステージノードをクリック
    │
    ▼
/learn/[chapterId] (ステージ進行 / 学習ワークスペース)
```

---

## 画面1: カリキュラム選択 (`/curriculum`)

### 概要

本棚UIでカリキュラムを選択する画面。RPGの「冒険の書」をモチーフにしている。

### アクセス条件

- 認証必須（未認証の場合 `/login` にリダイレクト）

### UI構成

| 要素           | 説明                                                      |
| -------------- | --------------------------------------------------------- |
| 背景           | 暗い暖色のグラデーション（図書館風）                      |
| タイトル       | 「冒険の書」（セリフ体、金色グロー）                      |
| 本棚           | 木目調の棚板にカリキュラムが本として並ぶ                  |
| 空きスペース   | 「Coming Soon」のプレースホルダー（将来のカリキュラム用） |
| ヒントテキスト | 「本をクリックして冒険を始めよう」                        |

### 本棚の仕様

- 最大5冊表示（現在1冊 + 4つの Coming Soon）
- 各本は背表紙で表示（縦書きタイトル、金色の装飾ライン）
- 本の色はカリキュラムごとに異なる（赤、青、緑、紫、琥珀）

### 本を開くアニメーション（4段階）

| 段階            | 時間       | 動作                                                           |
| --------------- | ---------- | -------------------------------------------------------------- |
| 1. 引き出し     | 0〜0.5秒   | 本が棚から上に浮き上がる                                       |
| 2. 表紙表示     | 0.5〜1.2秒 | 棚の本が消え、画面中央に表紙が現れる                           |
| 3. ページが開く | 1.2〜1.9秒 | 表紙が左に回転して開き、中のページが見える（デスクトップのみ） |
| 4. 完了         | 1.9秒〜    | 見開き状態で操作可能                                           |

### 開いた本の中身

**デスクトップ（md以上）: 見開き表示**

| 左ページ               | 右ページ                        |
| ---------------------- | ------------------------------- |
| Adventure Log ヘッダー | Status ヘッダー                 |
| カリキュラムタイトル   | 進捗バー（%表示）               |
| 説明文                 | 学習時間（未実装: --:--）       |
| ページ番号             | 最終プレイ日（未実装: ---）     |
|                        | 難易度（星評価）                |
|                        | 「冒険を始める / 続ける」ボタン |

**モバイル（md未満）: 縦1ページ表示**

- 上部: 表紙カラーのヘッダー（タイトル、説明）
- 下部: ステータス + 「冒険を始める / 続ける」ボタン
- フェードスライドアップアニメーション

### 遷移先

- 「冒険を始める / 続ける」ボタン → `/curriculum/[slug]`

## 画面2: ステージ選択 (`/curriculum/[slug]`)

### 概要

Duolingo風の縦スクロールステージマップ。ファンタジー背景画像の上にステージノードが蛇行パスで並ぶ。

### アクセス条件

- 認証必須
- 存在しないスラッグの場合 404

### UI構成

| 要素           | 説明                                              |
| -------------- | ------------------------------------------------- |
| 背景           | ファンタジー風景画像（PC用 / スマホ用で切り替え） |
| ヘッダー       | 戻るボタン + カリキュラムタイトル + 完了数        |
| プログレスバー | 金色グラデーション、完了ステージ数に応じた幅      |
| ステージマップ | 10ステージが蛇行パスで縦に並ぶ                    |

### 背景画像

| デバイス         | 画像                   | サイズ    |
| ---------------- | ---------------------- | --------- |
| PC（md以上）     | `/stage-bg.png`        | 1448x1086 |
| スマホ（md未満） | `/stage-bg-mobile.png` | 941x1672  |

### ステージノードの仕様

| ステータス | 見た目                                      | 操作                   |
| ---------- | ------------------------------------------- | ---------------------- |
| completed  | 金色グラデーション + チェックマーク + 星3つ | クリックで学習ページへ |
| current    | 緑グラデーション + パルスアニメーション     | クリックで学習ページへ |
| locked     | グレーグラデーション + 鍵アイコン + 半透明  | クリック不可           |

### ステージ一覧（ダミー10ステージ）

| #   | タイトル       | アイコン | 状態               |
| --- | -------------- | -------- | ------------------ |
| 1   | 背景画像       | ⭐       | DB連動（実装済み） |
| 2   | テキスト装飾   | 🎨       | ダミー             |
| 3   | ボックスモデル | 📐       | ダミー             |
| 4   | Flexbox基礎    | 🖼️       | ダミー             |
| 5   | Grid基礎       | 📦       | ダミー             |
| 6   | レスポンシブ   | 🧩       | ダミー             |
| 7   | アニメーション | 🎯       | ダミー             |
| 8   | フォーム設計   | 🚀       | ダミー             |
| 9   | レイアウト実践 | 💎       | ダミー             |
| 10  | 総合テスト     | 🏆       | ダミー             |

### パスコネクター

- 完了ステージ間: 金色グラデーション + グロー
- 未完了ステージ間: 白の半透明

### 蛇行パターン

オフセット: `[0, -60, 0, 60, 0, -60, 0, 60, 0, -60]`

### 遷移先

- ステージ1クリック → `/learn/background-image`
- ステージ2〜10クリック → アラート「準備中です」

## 画面3: ステージ進行 (`/learn/[chapterId]`)

### 概要

インタラクティブなコード学習ワークスペース。左に解説、右にコードエディタとプレビューが並ぶ。

### アクセス条件

- 認証不要（静的生成）
- 存在しないチャプターIDの場合 404

### チャプター構成

各チャプターは「レッスン（解説）」と「プラクティス（実践）」のステップで構成される。

### 現在のチャプター: `background-image`

| #   | 種類     | タイトル                                   |
| --- | -------- | ------------------------------------------ |
| 1   | lesson   | Webページの画像、2つの入れ方               |
| 2   | lesson   | どっちを使う？判断フローチャート           |
| 3   | lesson   | background-image の「4点セット」           |
| 4   | practice | background-image で画像を表示しよう        |
| 5   | practice | background-repeat で繰り返しを止めよう     |
| 6   | practice | background-size で要素いっぱいに広げよう   |
| 7   | practice | background-position で表示位置を調整しよう |
| 8   | practice | チェックポイント: 4点セットを全部書こう    |

### レッスンステップ（lesson）

- 解説のみ（コードエディタなし）
- 「前へ」「次のステップへ」ボタンで移動

### プラクティスステップ（practice）

**レイアウト**

| 左パネル（w-80）           | 右パネル（flex-1）              |
| -------------------------- | ------------------------------- |
| ステップバッジ             | コードエディタ（HTML/CSS タブ） |
| タイトル                   |                                 |
| 説明文                     | プレビュー（iframe）            |
| ヒントボタン               |                                 |
| 「前へ」「採点する」ボタン |                                 |

### ヒントモーダル

- 「ヒントを見る (N件)」ボタンで開く
- 前へ/次へボタンで1件ずつ閲覧
- ドットインジケーターで直接ジャンプ可能
- 最後のヒントでは「次へ」が「閉じる」に変わる

### 採点モーダル（3段階）

| 段階   | 表示                                                                     |
| ------ | ------------------------------------------------------------------------ |
| 採点中 | ドットが跳ねるアニメーション + 「採点中...」テキスト                     |
| 合格   | 緑のチェックマーク + 成功メッセージ + 「次のステップへ」ボタン           |
| 不合格 | 赤のバツ + 「確認してほしいポイント」リスト + 「コードを修正する」ボタン |

### 採点基準

`getComputedStyle` ベースでCSSプロパティ値を検証。

| 条件タイプ | 説明                           |
| ---------- | ------------------------------ |
| equals     | 完全一致                       |
| not-equals | 値が指定値でない、かつ空でない |
| includes   | 部分一致                       |

各ルールに `failureHint` を持ち、不合格時に具体的なフィードバックを表示。

### プログレスバー

- 画面上部に表示
- 現在のステップ / 全ステップ数

## データモデル

### DB テーブル（Supabase）

| テーブル            | 用途             |
| ------------------- | ---------------- |
| curriculums         | カリキュラム定義 |
| stages              | ステージ定義     |
| user_stage_progress | ユーザーの進捗   |
| reference_articles  | 参考記事         |

### フロントエンド型定義

```typescript
// ステージの状態
type StageStatus = 'completed' | 'in_progress' | 'unlocked' | 'locked';

// チェックルール
type CheckRule = {
  property: string;
  condition: 'not-equals' | 'equals' | 'includes';
  value: string;
  failureHint?: string;
};

// 採点結果
type GradeResult = {
  passed: boolean;
  failedHints: string[];
};

// チャプター構成
type Chapter = {
  id: string;
  title: string;
  steps: (LessonStep | PracticeStep)[];
};
```

## コンポーネント構成

```
/curriculum
  └── BookShelf (本棚 + 本を開くアニメーション)

/curriculum/[slug]
  ├── ParallaxBackground (背景画像)
  ├── StageMapClient (ステージデータ構築)
  └── DuolingoStageMap (ステージノード描画)

/learn/[chapterId]
  └── LearnWorkspace
      ├── ProgressBar
      ├── LessonView (解説ステップ)
      ├── StepGuide (実践ステップガイド)
      │   ├── HintModal
      │   └── GradeModal
      ├── CodeEditor (HTML/CSS エディタ)
      └── Preview (iframe プレビュー)
```
